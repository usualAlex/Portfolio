About the UI

A fully simulated diegetic UI.


Main menu screen (WebGL)

I challenged myself to create a full diegetic UI where the blocks used in levels also have UI functions like resetting level and going back. They are spawned and simulated with physics just like any other block. They are used as part of the level design and even help you.

Even the main menu has a reset button because you can interact with the blocks. You practically learn how to play the game from the very first menu you encounter.

While the web demo currently doesn't use keyboard inputs, besides quickly using "R" to reset a level you also have the reset button as a block very close to where the action happens. This limits the amount of mouse travel to reset a level or exit. Compared to using a standard UI with buttons in the far corners of the screen.


Settings screen (WebGL)

The settings screen was a little tricky to keep entirely as blocks. They are all fully simulated with physics.

Project Overview





Jex in Space is a physics-based mobile runner game with a one-touch recoil mechanic and time-looped death system. You shoot in order to move. Recoil pushes you back and spins you. So with one single tap: you aim, shoot and move (based on time reflex).





This little game is especially meaningful to me. In early 2024, I set out to learn Unity and C# so I could bring my own game ideas to life. Jex in Space was my first solo project, and it taught me not only a programming base, but also how UI gets implemented at engine level.

Looking back, I’m still amazed I built the entire game myself. Code, design, UI, animations, etc. I got a lot more appreciation for developers and sharpened my skills as a UI designer.



Project Challenges

- Designing a one-tap control that allows you to move, aim and shoot all in one input
- Creating an immersive UI that fits the sci-fi tone (augmented HUD and holographic console)
- Making the UI scale well across different screen sizes
- Balancing difficulty across multiple screens (PC vs Mobile - smaller screens, harder game)
- Designing and editing a game trailer with no prior video experience
- Designing an anthropomorphic cat character

Gameplay



You must help Jex survive hordes of rogue asteroids and space junk while adrift in space. Lose all your 9 lives and get thrown back in time to try again.

Trapped in a time bubble adds just a tad of roguelite feel to the game. You earn credits from destroying objects and use them to unlock weapons and upgrades.





Character Design

I wanted to design a cat character that was both cute and fun. Friendly appearence to appeal to a younger audience but suitable for any age. I'm not a character designer, therefore, I had a really hard time making something decent looking but i kept trying and eventually i got something that i liked.











Story

Centers around a pawlet (bracelet) that has been passed around the universe sold cheap at every space market. Some saying that it's an ancient device holding huge quantities of energy. Despite some studies conducted, no such properties were ever found. Kara, a debt-ridden pilot entangled in some space piracy and bounty hunting gets her hand on the pawlet and gives it to Jex, her cat. Recently an industrial planet has exploded creating chaos all around and... lots of valuable materials left to loot. Kara, Jex and a crew decide to take part and head towards the ex-planet. But the whole zone became a death-trap. Kara and the crew get caught in a wave of rogue asteroids proven to be impossible to avoid.

The ship gets hit!

The crew manages to run away using an escape pod. However, in all that rush Kara doesn’t realize until it’s too late: Jex is still on board. The ship explodes. The energy from that explosion activated the pawlet now used by Jex which saves his life while getting thown off the ship and adrift in space.

User Interface

The entire UI consists of an augmented HUD and a console, both provided by the pawlet. The Console as a holo projection and the HUD as a neural augmentation. This allows me to use the pawlet as an "excuse" for being able to pause the game - the pawlet has the power to stop the entropy around you.

Throughout the game i used small cat motifs like little ears you can notice on the HUD (middle top) or even the console top tabs look like cat ears.



HUD - Non-Diegetic UI (Helmet Augmentation)

While a helmet augmentation is a diegetic UI, i used it as a non-diegetic UI HUD overlay. Jex doesn't necessarily sees this, is just my representation of the HUD.




Bowl (as space helmet) - Diegetic UI

You don't have to take your eyes off your character to know if your health is low because the bowl will start cracking when you have less than 3 lives left. So you can focus on the character and gameplay when it's most important.

First crack tells you got 2 more lives left. Second crack tells you got 1 more life left. Then the bowl shatters. Also included a meta type of UI using a pulsing green or red screen glow when you gain or lose a life.







Holo Console - Diegetic UI




The problem

With the pawlet console is that this won't work on landscape on mobile. The holo is designed for a portrait screen and the paw would need to be way longer to keep the holo centered on the screen. I wanted to be able to scale the game on different screens/devices.



Solution

I created a handler script to check device type. Then i used the paw on mobile portrait and only a pawlet on bigger screens (tablets, desktops). This allows me to scale the game to different screen sizes while on mobile i only use portrait because it's the intended experience - a super easy one hand control.

I aimed for something immersive but extremely simple at its core and in terms of functionality and UX yet it was challenging to design if i wanted more scalability and maintain the immersive style. I'll keep this in mind for future games that maybe i want to play mobile-first games on PC (like using "Google Play Games on PC").






Progression

Guns and Upgrades to unlock

Multiple weapons you can unlock using earned credits. Each weapon having it's own unique projectile and upgrades such as projectile speed and cooling system.


Explosive Bullets - unlocked by default

Blue Plasma - a cold plasma that overheats slowly

Lasers - very fast with a normal overheat rate

Homing Rockets - easy to overheat but makes each target easy to destroy

Triple Red Plasma - hot, overheats fast but shoots 3 projectiles at once.

Type of objects you encounter

There are different types of objects constantly coming towards you. Each with different properties. They come at different sizes, different speeds, sometimes can get you overwhelmed by their number. Sometimes you must choose to lose one life in order to survive a wave.





Still some work left to do...

Currently fleshing out details, fixing small bugs and finishing the last polishing layer.

The goal and scope is to publish a fun little game you can jump into in seconds and play in quick sessions while waiting in line or on the bus. No penalties for quitting mid-session.
